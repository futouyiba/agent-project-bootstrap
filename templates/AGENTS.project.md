# Project coordination

- Use GitHub Issues and the repository Project as the mutable task source of truth.
- Read the linked Issue and its acceptance criteria before changing code.
- Do not expand scope silently; create or propose a follow-up Issue.
- Use a dedicated branch and pull request for each independently deliverable Issue.
- Report exact validation results and do not merge while required checks fail.
- Treat `.codex/cache/github-snapshot.json` as a disposable read-only cache, never as task state.
